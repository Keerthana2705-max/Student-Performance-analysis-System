  <h1>Student Performance Analysis System (Synthetic Data)</h1>
  <p>
Exploratory Data Analysis (EDA) Project</p>
<h1>Overview</h1>
<p>This project focuses on analyzing student academic performance using a synthetically generated dataset. Educational institutions often collect large volumes of student data, and manual analysis can be inefficient and error-prone.
This system automates the process of data generation, cleaning, analysis, and visualization to extract meaningful insights about student performance.</p>
<h1>The workflow includes:</h1>
  <p>
Synthetic data generation </p>
<p>Data preprocessing and cleaning </p>
<p>Exploratory Data Analysis (EDA) </p>
<p>Performance evaluation metrics</p> 
<p>Data visualization </p>
<p>Insight generation </p>
<p>The objective is to create a structured and scalable analytics workflow that helps identify trends, top performers, and students requiring improvement.</p>

<h1>Dataset</h1>
<p>Source: Synthetic (Randomly Generated Dataset)</p>
<h1>Description:</h1>
<p>The dataset consists of 250 student records with the following attributes:<p>
<p>Column Name	:Description</p>
<p>Student_ID:	Unique identifier</p>
<p>Name:	Student name</p>
<p>Gender:	Male / Female</p>
<p>Department:	CSE / IT / ECE</p>
<p>Year:	1st / 2nd / 3rd</p>
<p>Maths	Marks: (0–100)</p>
<p>Science	Marks :(0–100)</p>
<p>English	Marks: (0–100)</p>
<p>Attendance:	Percentage (50–100)</p>
<p>Internal Marks:	Marks (0–25)</p>

<h1>Objectives</h1>
<p>Generate a realistic synthetic dataset representing student performance </p>
<p>Perform data cleaning and preprocessing </p>
<p>Compute statistical measures (mean, median, standard deviation) </p>
<p>Analyze total marks, average marks, and grades </p>
<p>Identify top-performing and low-performing students </p>
<p>Perform group-based analysis: </p>
<p>oDepartment-wise performance</p> 
<p>oGender-wise comparison </p>
<p>oYear-wise trends </p>
<p>Study relationships between variables:</p> 
<p>oAttendance vs Performance </p>
<p>oInternal Marks vs Final Scores </p>
<p>Visualize data using charts and graphs </p>
<p>Derive actionable insights for academic improvement </p>
<h1>Project Highlights</h1>
<p>1. Data Preprocessing</p>
<p>Checked for missing values and data types </p>
<p>Ensured consistency in categorical data</p> 
<p>Structured dataset for analysis </p>
<p>Generated new features:</p> 
<p>oTotal Marks </p>
<p>oAverage Marks </p>
<p>oGrade Classification </p>
<p>2. Exploratory Data Analysis (EDA)</p>
Analyzed distribution of marks using histograms <br>
Studied relationships between variables using scatter plots <br>
Identified patterns in student performance <br>
Examined variability using statistical summaries <br>
<p>3. Performance Analysis</p>
Calculated:<br> 
oTotal Marks<br>
oAverage Marks <br>
<p>Assigned grades based on performance: </p>
oA (≥75) <br>
oB (60–74)<br> 
oC (50–59) <br>
oFail (<50)<br> 
<p>Identified:</p> 
oTop 10 students <br>
oStudents who failed <br>
<p>4. Group-Based Analysis</p>
Department-wise Performance: Compared average marks across CSE, IT, and ECE <br>
Gender-wise Analysis: Compared male and female performance <br>
Year-wise Trends: Observed performance differences across academic years <br>
<p>5. Visualization</p>
  <p>The project includes the following visualizations:</p>
Bar charts (Department performance) <br>
Pie charts (Gender distribution) <br>
Histograms (Marks distribution) <br>
Scatter plots (Attendance vs Marks) <br>
Heatmap (Correlation analysis) <br>

Internal marks positively influence final performance <br>
Certain departments show higher average performance <br>
Performance varies slightly across academic years <br>
A small percentage of students fall into the fail category<br> 
<p>Interpretation</p>
Attendance Impact: Strong correlation with academic success<br> 
Internal Marks Influence: Useful indicator of overall performance<br>
